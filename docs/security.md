# Security and data residency

## Hosted MCP App server

When using `https://mcp.draw.io/mcp`, the diagram payload is included in the MCP request and is processed by the hosted draw.io MCP server.

Use this mode for non-sensitive or approved content where the hosted service is within the acceptable trust boundary.

## Local MCP Tool Server

When using `npx -y @drawio/mcp`, diagram data is processed locally by the MCP process. The browser-opening workflow carries diagram content in the URL fragment, which browsers do not send to the remote web server as part of the HTTP request.

The draw.io web application and viewer assets may still be loaded from diagrams.net unless `DRAWIO_BASE_URL` points to a self-hosted draw.io deployment.

Example:

```jsonc
// .vscode/mcp.json
{
  "servers": {
    "drawio-local": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@drawio/mcp"],
      "env": { "DRAWIO_BASE_URL": "https://drawio.internal.example" }
    }
  }
}
```

## LLM boundary

Running the MCP server locally does not make the complete workflow local. Diagram definitions are generated by the selected LLM. For full isolation, use an approved locally hosted model and block or monitor network egress.

## Enterprise recommendation

Classify diagrams into three tiers:

- Public/reference: hosted MCP allowed.
- Internal: local MCP preferred.
- Restricted/confidential: local MCP, self-hosted draw.io, approved model endpoint, and controlled egress.
